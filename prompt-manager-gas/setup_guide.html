1
スプレッドシートの準備
データベースとなるスプシを作成します
1
Google スプレッドシートを新規作成する
▼

Google ドライブを開き、「新規」→「Google スプレッドシート」で新しいスプシを作成してください。

シート構成は以下の2シートを作成します（GASが自動作成するので手動作成は不要です）。

シート名用途
templatesテンプレートデータ（id / category / title / prompt / visible）
config管理者パスワード（admin_password）
📋 シートはGASが初回アクセス時に自動作成します。手動で作る必要はありません。
2
スプレッドシートIDを確認する
▼

スプレッドシートのURLから /d/ と /edit の間の文字列がIDです。

https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit ↑ この部分がスプレッドシートID

このIDは次のGAS設定で使います。メモしておいてください。

2
Google Apps Script の設定
GASをWebアプリとしてデプロイします
1
Apps Script を開く
▼

スプレッドシートのメニューから「拡張機能」→「Apps Script」をクリックしてください。

📌 メニューバー：拡張機能 → Apps Script

新しいタブでApps Scriptエディタが開きます。

2
コードを貼り付ける
▼

エディタに最初から書かれているコードをすべて削除して、gas_code.gs の内容をそのまま貼り付けてください。

⚠️ 既存の function myFunction(){} は必ず削除してから貼り付けてください。

貼り付けたら、1行目の SPREADSHEET_ID を自分のスプシIDに書き換えます。

// 変更前 const SPREADSHEET_ID = "ここにスプレッドシートIDを入力"; // 変更後（例） const SPREADSHEET_ID = "1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms";

変更したら 💾（保存）ボタンまたは Ctrl+S で保存してください。

3
Webアプリとしてデプロイする
▼

右上の「デプロイ」→「新しいデプロイ」をクリックしてください。

📌 デプロイ（右上青ボタン）→ 新しいデプロイ

設定項目を以下のように設定してください。

項目設定値
種類の選択ウェブアプリ
説明任意（例：プロンプト管理API）
次のユーザーとして実行自分
アクセスできるユーザー全員
⚠️ 「アクセスできるユーザー」は「全員」に設定してください。「自分のみ」だとHTMLからアクセスできません。

「デプロイ」ボタンをクリックするとGoogleアカウントの認証が求められます。「権限を確認」→「許可」を選択してください。

✓ デプロイが完了すると WebアプリのURL（https://script.google.com/macros/s/.../exec）が表示されます。必ずコピーしておいてください。
4
コード変更後の再デプロイについて
▼

GASのコードを変更した場合、「新しいデプロイ」ではなく「デプロイを管理」→ 既存のデプロイを編集（鉛筆アイコン）→「バージョン：新しいバージョン」で更新してください。

⚠️ 「新しいデプロイ」を毎回実行するとURLが変わってしまいます。既存デプロイの更新を使うとURLは変わりません。
3
GitHub Pages の設定
index.htmlをアップしてアプリを公開します
1
GitHubに新しいリポジトリを作成する
▼

github.com にログインし、右上「＋」→「New repository」で新しいリポジトリを作成してください。

項目設定値
Repository name例：prompt-manager-gas
公開設定Public
💡 GitHub Pagesを使うにはPublicリポジトリが必要です。
2
index.htmlをアップロードする
▼

作成したリポジトリページで「Add file」→「Upload files」をクリックし、index.html をドラッグ＆ドロップしてアップロードしてください。

📌 Add file → Upload files → index.html をドロップ → Commit changes
3
GitHub Pages を有効化する
▼

リポジトリの「Settings」タブ → 左メニュー「Pages」を開いてください。

項目設定値
SourceDeploy from a branch
Branchmain / (root)

「Save」をクリックすると数分後にURLが発行されます。

https://ユーザー名.github.io/リポジトリ名/
4
アプリの初期設定
GAS URLを登録して動作確認します
1
アプリを開いてログインする
▼

GitHub PagesのURLをブラウザで開いてください。初回はGAS URLが未設定のためログイン画面が表示されます。

初期パスワードは admin123 です。ログイン後すぐに変更することを推奨します。

⚠️ ローカルの index.html をダブルクリックで開いても動作しません。必ずGitHub PagesのURL（https://...）からアクセスしてください。
2
設定タブでGAS URLを登録する
▼

管理者でログイン後、右上の「設定」タブを開いてください。

📌 管理者画面 → 設定タブ → 「GAS WebアプリURL」欄

PHASE 2 でコピーしたGAS URLを貼り付けて「URLを保存して再接続」をクリックしてください。

✓ 「接続成功！テンプレート〇件を読み込みました」と表示されれば設定完了です。
3
パスワードを変更する
▼

設定タブの「管理者パスワードの変更」セクションでパスワードを変更してください。変更するとスプレッドシートの config シートに保存され、次回アクセス時から新しいパスワードが有効になります。

⚠️ 初期パスワード（admin123）のままにしておくとセキュリティ上危険です。必ず変更してください。
4
トラブルシューティング
▼
症状原因・対処
接続失敗と表示されるGAS URLが正しいか確認。デプロイ時の「アクセスできるユーザー」が「全員」になっているか確認
データが反映されないGASコードを変更した場合は「デプロイを管理」→既存デプロイを更新（新しいバージョン）する
ログインできない2回目以降はスプシのconfigシートのパスワードが使われる。スプシを直接確認して確認
前回のデータが出ないGAS URLはlocalStorageに保存される。別ブラウザ・別PCでは再設定が必要

🎉 セットアップ完了！

「ユーザー画面へ」ボタンで利用者向け画面を確認してください。
